# compliance/evidence/

Per-PR evidence packs auto-generated by BobGuard's `audit-evidence-generator` skill via the `/audit-pr` slash command.

## Structure

```
compliance/evidence/
  PR-{n}/
    control-map.md         ← markdown table of triggered controls
    data-flow.mmd          ← Mermaid source for ePHI flow
    data-flow.png          ← rendered diagram for the PDF
    threat-delta.md        ← attack-surface delta for this PR
    test-evidence.json     ← automated test results proving safeguards
    audit-pack.pdf         ← auditor-grade PDF (the hero artifact)
    governance-mock.json   ← only present if watsonx.governance API was unavailable
```

## How files get here

A developer opens a PR. From inside Bob IDE, in **Compliance Officer mode**, they run:

```
/audit-pr
```

The command (see `.bob/commands/audit-pr.md`) runs the full pipeline. Artifacts land here. A follow-up PR opens against `main` proposing to merge them.

`PR-1/` will be committed in Phase 5 as a working example, so judges browsing the repo see the full output without running anything.
